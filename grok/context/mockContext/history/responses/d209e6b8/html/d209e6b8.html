Does marked safely parse script tags embedded within markup text?

No, marked, a markdown parser, does not safely parse <script> tags embedded within markup text by default. When using marked to render markdown to HTML, any <script> tags included in the markdown text will be converted to HTML <script> tags. This can pose a security risk if the input is not sanitized, as it could lead to XSS (Cross-Site Scripting) attacks.

To safely handle <script> tags, you should use a separate sanitization library like DOMPurify after the markdown has been converted to HTML. This ensures that any potentially harmful scripts are removed or neutralized before the HTML is rendered in a web browser.
